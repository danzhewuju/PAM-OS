PAM-OS Memory 架构图

这张图把 Memory Runtime 拆成写入、升级、读取三条路径；重点是“普通记忆”如何经过 evidence 层，逐步升级成可解释、可注入的长期 profile trait。

Memory 升级怎么看

  • events 是原始证据，不应该直接变成“用户画像”。
  • memories 是可搜索事实，例如项目决策、偏好、目标、风格。
  • behavior_events 记录用户选择、拒绝、推迟什么，这是理解决策风格的关键入口。
  • profile_evidence 是升级缓冲层，说明某个长期判断从哪里来。
  • profile_traits 才是长期画像，会被 prepare_context 优先注入。

当前 v0 规则

  • 偏好、风格、目标、项目决策类 memory 可生成 evidence。
  • 行为选择里出现“选轻量本地方案、拒绝重型基础设施”会升级为 decision style。
  • 同一个 trait_key 会合并更新，提升 confidence 和 stability。
